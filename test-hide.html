🧪 Hide Parameter Test

What does hide=true do?

When hide=true is added to any URL, the AKA service should:

  • ✅ Suppress all console output and UI elements
  • ✅ Show only a white screen
  • ✅ Process redirects based on other URL parameters
  • ✅ Redirect immediately without user interaction

🚀 Test Links

Test 1: Root with hide=true (should show white screen)

/?hide=true
Test Root + Hide

Expected: White screen with no console output

Test 2: Root with hide=true and redirect parameter

/?hide=true&redirect=example
Test Root + Hide + Redirect

Expected: Silent redirect to target URL (if configured)

Test 3: Path with hide=true

/test?hide=true
Test Path + Hide

Expected: Silent redirect to target URL (if /test is configured)

Test 4: Path with hide=true and additional parameters

/docs?hide=true&version=latest
Test Path + Hide + Params

Expected: Silent redirect including the version=latest parameter

🔍 How to Test

  1. Open Browser DevTools (F12) before clicking test links
  2. Monitor Console - should be empty with hide=true
  3. Check Network Tab - should show minimal requests
  4. Observe Page - should show white screen or redirect immediately

📝 Testing Checklist

  • ❌ No console output (check DevTools Console)
  • ❌ White screen only (no UI elements visible)
  • ❌ Fast loading (no web component loading delays)
  • ❌ Immediate redirect (for valid routes)
  • ❌ Silent failures (no error pages)
